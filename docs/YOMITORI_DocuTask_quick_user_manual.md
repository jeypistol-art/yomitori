# YOMITORI DocuTask 簡易操作マニュアル

最終更新: 2026-06-09

YOMITORI DocuTaskは、PDF、画像、メール本文などの書類から、要約、期限、対応事項、タスク、リマインド、証跡を整理するための業務支援サービスです。

このマニュアルは、日常利用する担当者と、初期設定や運用管理を行う管理者向けの簡易版です。

## 1. 基本の流れ

日常業務では、次の順番で使います。

1. 書類を登録する
2. AI抽出結果を確認する
3. 必要に応じて内容を修正する
4. 承認してタスクを作成する
5. タスクとリマインドで対応を進める
6. 監査ログで操作履歴を確認する

AIの抽出結果は、最終確定情報ではありません。承認画面で人が確認してから、タスク化してください。

## 2. ログイン

ログイン画面から、次のいずれかの方法でログインします。

- Googleアカウントでログイン
- メールアドレス認証でログイン

メール認証の場合は、入力したメールアドレスに届くログインリンクからアクセスします。

## 3. ダッシュボード

ダッシュボードは、ログイン後に最初に見る画面です。

主に確認する項目は次の通りです。

- 次に対応すべき項目
- 承認待ち
- 未処理一覧
- 期限間近
- リマインド
- 最近の書類
- 今月の利用状況

画面上部の案内に、失敗書類、承認待ち、期限切れなどが表示されている場合は、そこから処理を始めると迷いにくくなります。

## 4. 初期設定

初期設定画面では、運用開始前に必要な項目を確認します。

主な確認項目は次の通りです。

- 管理対象の登録
- 取引先の登録
- 担当者の登録
- リマインド設定
- 外部サービス接続状況
- 未処理件数
- 失敗中ジョブ件数

異常が発生した場合は、初期設定画面の運用チェック項目を見て、問い合わせ時の確認材料として利用してください。

## 5. 台帳設定

台帳設定では、書類に紐づける管理対象と取引先を登録します。

### 5.1 管理対象

管理対象は、物件、施設、店舗、テナント、事務所など、書類の対象になる場所や契約対象を登録する項目です。

登録できる主な項目:

- 種別
- 名称
- コード
- 住所
- メモ
- 上位管理対象

コードは任意です。社内で使っている物件番号、施設ID、店舗コードなどがある場合に入力してください。

### 5.2 管理対象の種別追加

標準の種別に当てはまらない場合は、種別のプルダウンから「＋種別を追加」を選びます。

追加した種別は、追加済み種別として表示され、以後の登録時に選択できます。

不要になった追加種別は、追加済み種別の削除ボタンから削除できます。削除した場合、その種別を使っていた登録済みデータは「その他」に戻ります。

### 5.3 取引先

取引先は、行政、自治体、業者、保険会社、リース会社、テナントなど、書類の発行元や関係先を登録する項目です。

登録できる主な項目:

- 種別
- 名称
- 担当者
- メール
- 電話
- 住所
- メモ

### 5.4 取引先の種別追加

標準の種別に当てはまらない場合は、種別のプルダウンから「＋種別を追加」を選びます。

追加した種別は、追加済み種別として表示され、不要になった場合は削除できます。

削除した場合、その種別を使っていた登録済みデータは「その他」に戻ります。

## 6. 担当者管理

チーム運用では、担当者を登録しておくことで、AI抽出確認画面やタスク画面で担当者を割り当てられます。

担当者を登録しておくと、次の運用がしやすくなります。

- タスクの担当者割当
- リマインド通知
- 未割当タスクの確認
- 監査ログでの操作履歴確認

複数ユーザーや担当者割当は、プランによって利用可否が変わります。

## 7. 書類を登録

書類登録画面では、次の方法で書類を登録できます。

- PDFをアップロード
- 画像をアップロード
- メール本文や通知文を貼り付け

PDF・画像は、ファイル選択またはドラッグ＆ドロップで登録できます。

メール本文だけでも登録できます。自治体からのメール、業者からの更新案内、社内チャットで届いた依頼文などを貼り付けて利用できます。

### 7.1 登録時に設定できる項目

- タイトル
- 取引先
- 管理対象
- メール本文・通知文
- PDF・画像ファイル

タイトルを空欄にした場合、ファイル名や本文の先頭行から候補が入ることがあります。

### 7.2 台帳設定へ移動する場合

書類登録中に管理対象や取引先が未登録だった場合は、書類登録画面から台帳設定へ移動できます。

入力途中のタイトル、取引先、管理対象、本文は一時保存されるため、台帳設定後に書類登録画面へ戻っても、最初から入力し直す必要はありません。

## 8. AI抽出確認・承認

AI抽出確認・承認画面では、左側に原本、右側にAI抽出結果が表示されます。

主に確認する項目は次の通りです。

- 書類タイトル
- 書類種別
- 要約
- 重要事項
- 管理対象
- 期限
- 提出物
- タスク候補
- 注意点
- 過去書類との差分

内容に誤りがある場合は、この画面で修正してから承認します。

### 8.1 書類種別の追加

標準の書類種別に当てはまらない場合は、書類種別のプルダウンから「＋種別を追加」を選びます。

追加した書類種別は、追加済み書類種別として表示され、以後の承認画面でも選択できます。

不要になった追加書類種別は削除できます。削除した場合、その書類種別を使っていた登録済み書類は「その他」に戻ります。

### 8.2 管理対象の紐づけ

管理対象が未設定の場合、後から書類を探しにくくなる可能性があります。

承認前に、対象となる物件、施設、店舗、テナントなどを選択してください。

### 8.3 タスク候補

AIが抽出した対応事項は、タスク候補として表示されます。

タスク候補では、次の項目を確認します。

- タスク名
- 説明
- 期限
- 優先度
- 担当者
- リマインド日
- 承認時に作成するかどうか

不要なタスク候補は、承認時に作成しない設定にできます。

### 8.4 承認

内容を確認したら「承認してタスク作成」を押します。

承認すると、書類の状態が承認済みになり、作成対象のタスクがタスク一覧に追加されます。

## 9. 未処理一覧

未処理一覧では、まだ処理が必要な書類やタスクをまとめて確認できます。

主に確認する項目は次の通りです。

- AI未抽出
- 承認待ち
- 抽出失敗
- 未完了タスク
- 期限超過
- 担当未設定

優先処理が有効な場合は、期限切れ、期限接近、注意点ありなどをもとに、重要度順に表示されます。

## 10. タスク一覧

タスク一覧では、承認時に作成された対応タスクを確認・更新できます。

主な操作:

- ステータス変更
- 担当者変更
- 期限変更
- 優先度変更
- タスク削除

期限切れ、本日期限、今週対応、担当者未設定などの条件で確認できます。

## 11. リマインド

リマインド画面では、タスクに紐づいた通知予定や送信結果を確認できます。

主な確認項目:

- 全て
- 期限切れ
- 本日
- 予定リマインド
- 送信失敗

リマインドメールは、設定されたタイミングで担当者へ送信されます。

タスクが完了済みになった場合、そのタスクに対する未送信リマインドは送信対象から外れます。

## 12. 監査ログ

監査ログでは、主な操作履歴を確認できます。

記録される主な内容:

- 書類承認
- タスク作成
- リマインド送信
- リマインド再送
- 課金・プラン関連の変更
- API/Webhook関連の操作

誰が、いつ、何を行ったかを確認したい場合に利用します。

## 13. 利用状況・プラン

利用状況・プラン画面では、現在のプラン、今月の利用状況、追加パック、プラン別の機能差分を確認できます。

主な確認項目:

- 現在のプラン
- 今月の書類登録数
- 残り登録可能数
- 追加パック
- プラン変更
- 解約予定
- 機能差分

利用できない機能はグレーアウトされ、対象プランの案内が表示されます。

## 14. API/Webhook

API/Webhookは、YOMITORI DocuTaskで解析・登録した書類やタスクを、外部システムに連携するための機能です。

一般利用者が日常的に操作する画面ではありません。運用する場合は、システム管理者と相談してください。

主な用途:

- 外部システムから書類一覧を参照する
- 外部システムからタスク一覧を参照する
- 書類承認やタスク作成を外部システムへ通知する
- Webhookの送信履歴を確認する

## 15. よくある操作

### 15.1 書類を登録してタスク化する

1. ダッシュボードから「書類を登録」を開く
2. PDF、画像、またはメール本文を登録する
3. AI抽出を実行する
4. 「確認」からAI抽出確認・承認画面を開く
5. 書類タイトル、種別、要約、期限、タスク候補を確認する
6. 必要に応じて修正する
7. 「承認してタスク作成」を押す
8. タスク一覧または未処理一覧で作成されたタスクを確認する

### 15.2 管理対象や取引先を後から追加する

1. 書類登録画面で「台帳設定」または「取引先設定」を開く
2. 管理対象または取引先を追加する
3. 「書類登録へ戻る」を押す
4. 入力途中の内容が復元されていることを確認する
5. 登録を続ける

### 15.3 期限切れタスクを確認する

1. ダッシュボードで期限切れまたは期限間近の案内を見る
2. タスク一覧または未処理一覧を開く
3. 対象タスクの内容、期限、担当者を確認する
4. 対応が終わったらステータスを完了にする

### 15.4 送信失敗したリマインドを確認する

1. ダッシュボードまたはリマインド画面を開く
2. 送信失敗のリマインドを確認する
3. 担当者メールアドレス、送信元設定、Resend設定を確認する
4. 必要に応じて再送する

## 16. 注意事項

- AI抽出結果は必ず人が確認してください。
- 期限や提出物は、原本と照らし合わせて確認してください。
- 書類種別、管理対象、取引先を設定しておくと、後から検索・確認しやすくなります。
- 担当者未設定のタスクは通知や引き継ぎが漏れやすいため、早めに割り当ててください。
- 不要な追加種別を削除すると、その種別を使っていたデータは「その他」に戻ります。
- API/Webhookは外部システム連携用です。設定変更はシステム管理者が行ってください。

## 17. 問い合わせ時に伝えるとよい情報

問い合わせ時は、次の情報があると原因を確認しやすくなります。

- 発生した画面名
- 操作した日時
- 書類IDまたはタスクID
- 表示されたエラーメッセージ
- 登録したファイル形式
- 対象の管理対象または取引先
- 利用中のプラン
- 初期設定画面の運用チェック結果

書類IDは、AI抽出確認・承認画面のURLに含まれるUUIDです。
