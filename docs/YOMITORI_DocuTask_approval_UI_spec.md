# YOMITORI DocuTask 承認画面 詳細UI仕様

## 1. 目的

承認画面は、AIが抽出した内容を人間が確認・修正・承認し、正式な書類台帳・タスク・リマインドへ反映するための中核画面。

この画面で解決すること:

- 原本を読みに行く時間を減らす
- AI抽出結果の誤りを発見しやすくする
- 期限・提出物・担当者の見落としを防ぐ
- 承認者と承認日時を証跡として残す
- 承認後にタスクとリマインドを作成する

## 2. 基本方針

### AIの扱い

AI抽出結果は候補であり、自動確定しない。

ユーザーが承認した時点で、以下に正式反映する。

- documents
- tasks
- reminders
- document_approvals
- audit_logs

### 画面思想

左に原本、右に抽出結果を置く2ペイン構造にする。

原本を見ながら、右側のフォームで確認・修正・承認できることを最優先する。

## 3. 画面名

AI抽出確認・承認

URL例:

```text
/documents/:documentId/review
```

## 4. 利用ユーザー

### owner / admin

- 抽出結果を確認できる
- 抽出結果を修正できる
- 承認できる
- 再解析できる
- タスクを作成できる

### member

- 割り当てられた書類を確認できる
- 抽出結果を修正できる
- 権限設定により承認できる

### viewer

- 原本と抽出結果を閲覧できる
- 承認・修正はできない

MVPでは owner / admin / member を中心に実装し、viewerの細かい制御は後続でもよい。

## 5. 画面全体レイアウト

### デスクトップ

```text
┌────────────────────────────────────────────────────────────────────┐
│ Header: 書類名 / ステータス / 操作ボタン                            │
├───────────────────────────────┬────────────────────────────────────┤
│ 左ペイン: 原本プレビュー        │ 右ペイン: AI抽出結果フォーム          │
│                               │                                    │
│ PDF/画像                       │ 書類概要                           │
│ ページ操作                     │ 重要事項                           │
│ 拡大縮小                       │ 期限・提出物                       │
│ ハイライト                     │ タスク候補                         │
│ OCRテキスト                    │ リマインド                         │
│                               │ 承認・修正・再解析                  │
└───────────────────────────────┴────────────────────────────────────┘
```

### 推奨比率

- 左ペイン: 56%
- 右ペイン: 44%

右ペインはフォーム量が多いため、最小幅は 420px 程度を確保する。

### モバイル

MVPでは、承認作業はPCまたはタブレットを主対象にする。

スマホではタブ切替にする。

- 原本タブ
- 抽出結果タブ

スマホでの承認は可能にするが、業務上はPC推奨とする。

## 7. 左ペイン: 原本プレビュー

### 7.1 PDF / 画像ビューア

必須機能:

- PDF表示
- 画像表示
- ページ切り替え
- 拡大
- 縮小
- 幅に合わせる
- 90度回転
- ページ番号表示
- 全ページサムネイル

MVPでは、PDF.js相当のビューアまたは既存の実装資産を使う。

### 7.2 ハイライト表示

AIが抽出した項目に source_ref がある場合、原本上にハイライトを表示する。

ハイライト対象:

- 期限
- 提出物
- 必要対応
- 金額
- 契約期間
- 解約申出期限
- 問い合わせ先
- リスク・注意点

### 7.3 ハイライト操作

右ペインの項目をクリックすると、左ペインの該当ページに移動してハイライトする。

動作:

1. 右ペインの「期限」項目をクリック
2. 左ペインが根拠ページへ移動
3. 該当箇所をハイライト
4. 一時的にハイライトを強調表示

### 7.4 OCRテキスト表示

原本プレビュー下部またはサイドタブにOCRテキストを表示できるようにする。

用途:

- OCR結果の確認
- 原本が読みにくい場合の補助
- AI抽出根拠の確認

初期UIでは折りたたみ表示でよい。

### 7.5 エラー表示

表示できない場合:

- ファイル読み込みエラー
- 署名付きURL期限切れ
- PDF破損
- 画像形式非対応

ユーザーに出す文言:

```text
原本を表示できませんでした。再読み込みするか、ファイルを再登録してください。
```

## 9. 要確認アラート

AIの信頼度や不足項目に基づいて、画面上部に警告を表示する。

### 表示条件

- 主期限が抽出できない
- 期限候補が複数ある
- 提出物があるが期限がない
- 管理対象が未設定
- 担当者が未設定
- confidence が低い
- 契約更新書類なのに更新期限がない
- 解約申出期限がある
- 「至急」「必ず」「提出」「期限」などの重要語がある

### 表示例

```text
要確認: 期限候補が2件あります。主期限を選択してください。
```

```text
要確認: 解約申出期限が抽出されています。更新判断のタスク化を推奨します。
```

## 13. やること・タスク候補

- 説明

- 期限

- リマインド

- 根拠

### タスク候補の操作

- 追加
- 削除
- 複製
- 担当者変更
- 期限変更
- 優先度変更
- リマインド変更
- 根拠箇所へ移動

### 優先度

- 低
- 通常

- 至急

### デフォルトルール

- 期限が7日以内: 高
- 期限が3日以内: 至急
- 解約申出期限: 高
- 担当者未設定: 承認前警告

## 14. 担当者・リマインド

### 担当者

担当者候補はAIが出してよいが、最終選択はユーザーが行う。

UI:

- メンバー選択
- 未設定
- 新規メンバー招待導線

### リマインド候補

初期候補:

- 期限7日前
- 期限3日前
- 期限前日
- 期限当日

期限が近い場合:

- 期限3日前を過ぎている場合は、翌営業日または即時通知を候補にする

MVPでは営業日計算は簡易でよい。

## 15. 取引先・管理対象

### 取引先

AIが発行元・契約相手を推定する。

UI:

- 既存取引先選択
- 新規取引先作成
- 未設定

### 管理対象

AIが対象物件・施設・テナントを推定する。

UI:

- 複数選択
- 新規管理対象作成
- 未設定

### 管理対象が未設定の場合

承認自体は可能にするが、管理会社向け価値の中心なので警告を出す。

```text
管理対象が未設定です。後から台帳で探しにくくなる可能性があります。
```

## 16. 返信文案・メモ

### 返信文案

AIが必要に応じて返信文案を生成する。

表示条件:

- 回答期限がある
- 問い合わせ先がある
- 申請・提出・更新判断が必要

操作:

- コピー
- 編集
- 非表示

### 内部メモ

ユーザーが書類にメモを残せる。

メモはタスクコメントとは別に、書類台帳側のメモとして扱う。

## 17. 承認操作

### ボタン

- 下書き保存
- 承認してタスク作成
- 承認のみ
- 再解析
- 差し戻し

### 推奨メインCTA

```text
承認してタスク作成
```

タスク候補がない場合は:

```text
承認して台帳に保存
```

### 承認前チェック

承認ボタン押下時に以下を確認する。

- 書類タイトルがある
- 書類種別がある
- 必要対応がある場合、タスク候補がある
- タスク候補に期限がある
- タスク候補に担当者がある
- 期限が過去日でない
- 管理対象が設定されている
- 低confidenceの重要項目を確認済みにしている

### 警告とブロック

ブロックするもの:

- 書類タイトルなし
- 書類種別なし
- タスク作成対象なのにタスク名なし

警告して続行可能:

- 担当者未設定
- 主期限未設定
- 管理対象未設定
- confidence低
- 期限が過去日

## 18. 再解析

### 再解析の用途

- OCR結果が悪い
- 書類種別を変更した
- 管理対象や取引先を指定して解析し直したい
- AI抽出結果が大きく外れている

### 再解析前の確認

```text
再解析すると、現在のAI抽出候補が新しい結果に置き換わります。手動修正した内容は下書きとして保持されます。
```

### 保存方針

- ai_extractions は新しいレコードを作る
- 古い ai_extractions は superseded 扱い
- ユーザーの手動修正は draft として保持
- 承認済み内容は上書きしない

## 20. 保存タイミング

### 自動保存

右ペインの編集内容は一定間隔で自動保存する。

推奨:

- 3秒 debounce
- 画面離脱時に保存確認

保存先:

- 承認前の編集内容は review_draft として保存
- 正式な documents / tasks への反映は承認時

MVPでは review_draft テーブルを作らず、documents に draft_json を追加する案もある。ただしAI抽出世代や再解析が増えるため、後続では独立テーブル化が望ましい。

## 21. 承認時のデータ反映

承認時に行う処理:

1. documents を承認済み内容で更新
2. document_assets を更新
3. counterparties を紐づけ
4. tasks を作成
5. reminders を作成
6. document_approvals を作成
7. usage_events は既に消費済みなら更新しない
8. audit_logs を作成
9. notifications を作成

### トランザクション

承認処理はDBトランザクションで行う。

途中で失敗した場合:

- tasksだけ作られる状態を避ける
- remindersだけ作られる状態を避ける
- document_approvalsだけ作られる状態を避ける

## 22. キーボード操作

MVPで対応したいもの:

- Ctrl + S: 下書き保存
- Esc: モーダルを閉じる
- Tab: フォーム移動
- Enter: セレクト確定

後続候補:

- Ctrl + Enter: 承認
- F: 原本幅に合わせる
- +/-: ズーム

## 23. アクセシビリティ

- フォーム項目にはlabelを設定する
- ボタンはアイコンだけにしない
- エラーは色だけで示さない
- ハイライトにはテキスト説明を付ける
- キーボード操作で承認まで到達できる

## 25. API要件

### 取得

```text
GET /documents/:id/review
```

返すもの:

- document
- document_files
- document_pages
- latest_ai_extraction
- extracted_items
- review_draft
- task_candidates
- members
- managed_assets
- counterparties

### 下書き保存

```text
PATCH /documents/:id/review-draft
```

保存するもの:

- 編集中の抽出結果
- タスク候補
- 担当者
- リマインド
- 確認済みフラグ

### 承認

```text
POST /documents/:id/approve
```

送信するもの:

- approved_document
- accepted_items
- task_creations
- reminder_creations
- approval_comment

### 再解析

```text
POST /documents/:id/reprocess
```

送信するもの:

- document_type
- managed_asset_ids
- counterparty_id
- user_instruction

## 26. AI抽出JSONとの対応

承認画面の右ペインは、AI抽出JSONの以下を表示・編集する。

| UIセクション | JSONパス |
|---|---|
| 書類基本情報 | document_classification / document_summary / parties |
| 要約・重要事項 | document_summary.short_summary / document_summary.key_points |
| 期限 | important_dates |
| 提出物 | required_documents |
| やること | required_actions |
| タスク候補 | task_candidates |
| リマインド候補 | reminder_candidates |
| 取引先 | parties |
| 管理対象 | managed_asset_candidates |
| 注意点・リスク | risks_and_notes |
| 問い合わせ先 | contact_points |
| 返信文案 | reply_draft |
| 保管名・タグ | storage_suggestion |
| 要確認アラート | warnings / missing_information / quality |

右ペインの項目クリック時は、各項目の source_refs を参照して左ペインの該当ページ・該当箇所を表示する。

承認時は、編集済みの右ペイン内容を approved_document / task_creations / reminder_creations に変換して送信する。

## 27. UI受け入れ条件

MVPでの合格ライン:

- 原本と抽出結果を同時に確認できる
- AI抽出結果をフォームで修正できる
- 期限・提出物・タスク候補を確認できる
- 担当者とリマインドを設定できる
- 承認すると台帳とタスクに反映される
- 承認者と承認日時が残る
- 未処理一覧からこの画面へ遷移できる
- 承認後にタスク詳細へ遷移できる
